# SWVA Can Code Website - Project Documentation

## 1) Purpose

This repository contains the public-facing SWVA Can Code website, including:
- Main informational pages (Home, About, Contact, etc.)
- Student showcase content
- Sponsor information and sponsorship packet access
- Curriculum information and downloadable curriculum PDFs
- Internal UX documentation routes under `/docs`

---

## 2) Technology Stack

- Build tool: Vite
- Frontend framework: React
- Language: TypeScript
- Routing: React Router (`createBrowserRouter`)
- Styling: Tailwind CSS utility classes
- Icons: lucide-react

---

## 3) Run and Build

From the project root:

- Install dependencies: `npm i`
- Start development server: `npm run dev`
- Build production output: `npm run build`
- Preview production build: `npm run preview`

Build artifacts are generated into `/build`.

---

## 4) Repository Structure

- `/src` - application source code
  - `/src/pages` - route-level pages for website visitors
  - `/src/components` - shared layout/content components and docs-route components
  - `/src/data` - editable page content objects (preferred place for text/list updates)
  - `/src/assets` - local static assets used by bundler (images, logos, sponsor packet)
  - `/src/styles` - additional style files (if used)
  - `/src/lib` - utility folder (currently minimal)
- `/public` - static files served directly (including curriculum PDFs)
- `/build` - generated production output (do not edit directly)
- `/README.md` - setup overview
- `/PROJECT-DOCUMENTATION.md` - this complete reference
- `/src/EDITING-GUIDE.md` - non-technical editing instructions

## 8) Asset and File Management

### Website image assets
- Main app assets: `/src/assets`

### Curriculum PDFs
- Source/public delivery path: `/public/curriculum`

Do not manually edit files inside `/build`; they are regenerated by `npm run build`.

---

## 9) Sponsor Page Content Management

All Sponsor page content is controlled from `/src/data/sponsorContent.ts`.

### Sponsor tier cards
- Edit: `tiers.items`
- Update title, amount, subtitle, and benefits text.

### Current sponsor logo grid
- Edit: `sponsors.items`
- Each item uses:
  - `name`: display/alt label
  - `fileName`: exact file name from `/src/assets/sponsor logo`

Only sponsors listed in `sponsors.items` are shown. This removes empty placeholder slots.

### Sponsorship packet file
- Edit: `packet.pdfFileName`
- Must match an existing file in `/src/assets/sponsor logo`
- The Sponsor page button opens this PDF in a new tab.

## 10) Non-Technical Editing Workflow

Recommended process for safe content updates:

1. Open the matching content file in `/src/data`.
2. Edit text/array entries only.
3. If adding assets, place files in the expected folder first.
4. Run `npm run dev` to preview changes.
5. Run `npm run build` before publishing.

For step-by-step beginner instructions, use `/src/EDITING-GUIDE.md`.

## 12) Maintenance Notes

- Prefer editing `src/data/*.ts` for copy/content changes.
- Keep component logic changes minimal unless needed for new behavior.
- Keep image names readable and stable to simplify future content edits.
- Validate with `npm run build` after structural changes.
